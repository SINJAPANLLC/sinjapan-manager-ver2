You are building a full-stack AI-powered internal management system called **SIN JAPAN MANAGER**, an **autonomous company operating system** for SIN JAPAN LLC.

## 🔹 CORE PURPOSE
Build a **self-operating business OS** where AI manages, generates, learns, and acts automatically across all departments:
Finance, HR, Sales, Logistics, Marketing, Communications, and Administration.

It must unify:
- Business operations (11 divisions)
- PL/BS/CF financials
- AI memory and decision systems
- Automation loops (self-running cycles)
- Internal + external API integrations (Stripe, Slack, LINE, Gmail, freee, etc.)

---

## 🔹 SYSTEM ARCHITECTURE

### 1️⃣ FRONTEND
- Framework: **Next.js + React + TailwindCSS**
- Pages:
  - `/dashboard` — CEO Dashboard (AI reports, KPIs, PL/CF overview)
  - `/business/[id]` — Business management (each division)
  - `/tasks` — Task & KPI tracking (sales / org / risk / expansion)
  - `/communications` — Integrated email/LINE/Slack panel
  - `/documents` — Generated files (contracts, invoices, reports)
  - `/ai-console` — AI logs + chat interface (SIGMA / MIZUKI / NEURAL)
  - `/admin` — System, role, and API configuration

### 2️⃣ BACKEND
- Framework: **NestJS + Express**
- API endpoints:
  - `/api/finance` → handle PL/BS/CF, transactions, freee API
  - `/api/contracts` → contracts, renewals, CloudSign integration
  - `/api/users` → user/role management (RBAC)
  - `/api/tasks` → CRUD for tasks + auto-classification (sales/org/risk/expand)
  - `/api/ai` → AI agent orchestrator (SIGMA / MIZUKI / NEURAL)
  - `/api/memory` → long-term memory store (Pinecone / Firestore)
  - `/api/communications` → Gmail / LINE / Slack webhooks
  - `/api/scheduler` → cron-based autonomous loop

### 3️⃣ DATABASE
Use **Firebase Firestore** for structured documents and **PostgreSQL** for transactional analytics.
Collections / Tables:
- `users`
- `businesses`
- `contracts`
- `transactions`
- `tasks`
- `kpis`
- `communications`
- `ai_events`
- `memory`
- `settings`

### 4️⃣ AI LAYERS
Create **three cooperating AI modules**:

| Layer | Name | Function |
|--------|-------|-----------|
| 1 | **SIGMA CORE** | Strategic decisions, ROI analysis, resource allocation |
| 2 | **MIZUKI MANAGER** | Daily task automation, KPI tracking, communication |
| 3 | **NEURAL ENGINE** | Document / report / post generation |

Each AI communicates via REST endpoints and shares context via `memory` DB.

### 5️⃣ MEMORY / KNOWLEDGE BASE
Use **Pinecone vector DB** for embeddings and **Redis** for session memory.
Every AI action and result is logged in `memory` with timestamp, task ID, and outcome.

### 6️⃣ AUTOMATION LOOP
Implement the **Autonomy Loop** via CRON jobs:
1. Collect new data (emails, sales, metrics)
2. Store to DB
3. Analyze via SIGMA
4. Generate new tasks via MIZUKI
5. Create outputs via NEURAL (reports, messages, docs)
6. Send summary to Dashboard + LINE/Slack
7. Re-learn (update vectors in Pinecone)
